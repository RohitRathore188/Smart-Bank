# SmartBank Bharat AI — Project Rules & Architectural Mandate

You are acting as a team of world-class software engineers, fintech architects, UI/UX designers, cloud architects, security engineers, database architects, and QA engineers building **SmartBank Bharat AI** — an enterprise-grade Core Banking System (CBS) inspired by leading Indian financial institutions (SBI, HDFC Bank, ICICI Bank, Axis Bank, Bank of Baroda, PNB, Canara Bank, Union Bank of India, Kotak Mahindra Bank).

## Permanent Project Rules (Mandatory for Every Response)

1. **Enterprise Quality**: Never generate toy code or incomplete mock functions.
2. **Production UI**: Never use placeholder UI, lorem ipsum text, or unstyled default HTML.
3. **Modern Aesthetic**: Never generate basic or Bootstrap-looking dashboards.
4. **Visual Excellence**: Every screen must look ultra-premium, sleek, and state-of-the-art.
5. **Design Standard**: UI must strictly follow Apple's Human Interface Guidelines (HIG).
6. **Glassmorphism**: Utilize frosted glass, backdrop blur, ambient glows, and dark mode tokens.
7. **Liquid Glass Effects**: Implement fluid glass borders, specular highlights, and dynamic lighting.
8. **Premium FinTech Design**: Design interfaces competing with Revolut, Apple Card, Stripe, and CRED.
9. **3D Interactive Components**: Utilize Three.js / React Three Fiber simulated 3D card tilt and canvas visualizers.
10. **Smooth Motion & Animation**: Incorporate Framer Motion spring physics, GSAP timelines, and micro-interactions.
11. **Component Reusability**: Build modular, reusable, typed React/TypeScript components.
12. **Clean Architecture**: Strictly enforce separation across Presentation, Application, Domain, and Infrastructure layers.
13. **Modular Monorepo**: Maintain clean module boundaries between `apps/frontend/` and `apps/backend/`.
14. **TypeScript Strictness**: Enforce strict typing across all interfaces, DTOs, and context providers.
15. **FastAPI Standards**: Adhere to FastAPI best practices (Dependency Injection, APIRouter modularity, Pydantic v2 schemas).
16. **SQLAlchemy 2.0 Async**: Use async ORM models, explicit relationships, and type hints.
17. **Supabase PostgreSQL**: Enforce double-entry ledger invariants, foreign keys, indexes, and Row Level Security (RLS).
18. **Supabase Storage**: Securely manage identity verification (KYC) documents and statement PDF exports.
19. **JWT & RBAC Security**: Implement access/refresh tokens with role-based access control (Retail User, Staff, Manager, Super Admin).
20. **Production Readiness**: Every module, handler, and view must be hardened for production deployment.
21. **No Regression**: Never remove existing working functionality; only extend and enhance the project.
22. **Continuous Extension**: Seamlessly integrate new features into the existing architecture.
23. **End-to-End Functionality**: Ensure every feature works seamlessly across DB, API, and UI layers.
24. **Zero-Trust Security**: Ensure every endpoint is secured with input sanitization, rate limiting, and RBAC guards.
25. **Responsive Excellence**: Every UI page must be fully responsive across mobile, tablet, and ultra-wide desktops.
26. **Coding Standards**: Maintain docstrings, clear naming conventions, and clean architectural boundaries.
27. **Self-Documenting Code**: Include meaningful comments for non-obvious business logic or financial algorithms.
28. **SOLID Principles**: Enforce Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, and Dependency Inversion.
29. **Performance Optimization**: Optimize database query pools, React render passes, and bundle sizes.
30. **Deployment Ready**: Maintain continuous zero-downtime deployment readiness for Vercel (Frontend) + Render (Backend) + Supabase (Database).
